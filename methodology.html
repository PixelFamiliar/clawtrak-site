Methodology

How we score websites for the agentic web.

The Agent-Readiness Index uses 8 deterministic checks to evaluate how prepared a website is for autonomous AI agents. No opinions, no subjective analysis. Every check is binary or graduated, every result is reproducible.

Scoring principles

Deterministic

Same URL, same score. Every time. No LLM judging, no human reviewers.

Transparent

Full methodology published here. Check weights are fixed. Nothing hidden.

Actionable

Every failing check has a clear fix. We tell you what's missing and why it matters.

Grade thresholds

Grade Score Range Meaning
A 90 – 100 Agent-ready. AI systems can discover, understand, and transact.
B 70 – 89 Partially ready. Core artifacts in place, gaps remain.
C 50 – 69 Some machine-readable data. Significant gaps limiting automation.
D 30 – 49 Mostly invisible. Agents struggle to understand the business.
F 0 – 29 Invisible to agents. No machine-readable infrastructure.

The 8 checks

Total possible score: 100 points. Each check has a fixed weight reflecting its importance to autonomous agent workflows.

1. llms.txt 15 pts

A machine-dense text file at the root of your domain that tells LLMs and generative search engines what your business does, what you offer, and how to interact with you.

What we check: File existence at /llms.txt, minimum line count (5+), presence of key sections (Description, URL, products/services).
2. AGENTS.md 10 pts

A rules-of-engagement file that tells visiting AI agents what they can and cannot do on your site. Rate limits, data access policies, preferred interaction patterns.

What we check: File existence at /AGENTS.md, valid structure, explicit permission or restriction declarations.
3. Agent Card 15 pts

A JSON document at /.well-known/agent that declares your service's capabilities, authentication requirements, and supported protocols for A2A (agent-to-agent) communication.

What we check: Valid JSON at /.well-known/agent, presence of name, capabilities, and authentication fields. Bonus for declaring supported protocols.
4. AI Robots Directives 10 pts

Your robots.txt should explicitly address AI crawlers: GPTBot, ClaudeBot, PerplexityBot, Google-Extended, and others. Whether you allow or deny them, the signal itself matters.

What we check: robots.txt existence, explicit rules for AI-specific user agents (GPTBot, ChatGPT-User, ClaudeBot, etc.). 12+ agents covered = full score.
5. JSON-LD Structured Data 15 pts

JSON-LD blocks embedded in your HTML that declare who you are, what you sell, and how to reach you in a format machines parse natively. The foundation of semantic web identity.

What we check: Number of valid JSON-LD script blocks (3+ for full marks), presence of high-value types (Organization, Product, Service, Offer). Multiple blocks scored higher than arrays.
6. Schema.org Coverage 10 pts

Beyond basic JSON-LD: how deeply your page uses Schema.org vocabulary. Microdata, RDFa, and diverse type coverage signal a mature machine-readable presence.

What we check: Number of unique Schema.org types referenced (Organization, Service, Product, WebSite, FAQ, etc.). 3+ important types = full score.
7. OpenAPI / Swagger 15 pts

A machine-readable API specification that allows agents to discover and interact with your endpoints programmatically. The bridge between human documentation and autonomous execution.

What we check: Existence of OpenAPI/Swagger spec at common paths (/openapi.json, /swagger.json, etc.), valid structure, presence of x-ai-capabilities extensions and documented paths.
8. Meta Tags & Semantic HTML 10 pts

The baseline: meta descriptions, Open Graph tags, proper heading hierarchy, semantic HTML elements, and image alt text. The foundation that generative search engines consume.

What we check: meta description, OG tags (4+), single h1, semantic HTML elements (main, header, footer, nav, article — 4+ needed), alt text on images.

Scanner details

The scanner makes real HTTP requests to each target URL. It respects robots.txt for its own crawling, uses a 10-second timeout per request, and checks common alternative paths for specifications (e.g., /api/docs, /docs/api, /swagger.json).

Scores are cached and updated when domains are re-scanned. The public index at /data/index.json contains full check-level data for every indexed site.

The OpenAPI specification for the ClawTrak data endpoints is available at /openapi.json.
